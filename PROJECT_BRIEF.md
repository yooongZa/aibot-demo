# 프로젝트 브리프 (Project Brief)

> 이 문서는 챗봇·랜딩 사이트의 **톤·제안·업셀 카피**를 만들 때 근거가 되는 사업 컨텍스트 메모입니다.
> 향후 프롬프트·UX 변경 시 이 문서를 먼저 참고해서 사업 모델과 일관되게 유지하세요.

## 1. 사업 개요

- **사업명**: 개인 맞춤형 건강기능식품 배송 서비스
- **한 줄 정의**: "건강을 위해 낭비되는 돈·시간을 절약한다" — 식약처 고시 기능성 원료 근거 + 1일 권장 섭취량 최적화 + 전용 디스펜서 기반의 개인화 건기식 구독
- **사업 모델**
    - **1:1 AI 상담**: 소비자가 현재 섭취 중인 건기식을 분석 → 과용 성분, 불필요한 부원료, 비용 비효율, 추가 니즈를 진단하고 맞춤 처방을 제안
    - **영양소 최적화**: 식약처 1일 권장 섭취량 기준으로 함량·조합을 맞추고, 불필요한 부원료를 제외 (기능성 인정 성분만 사용)
    - **합리적 가격 구조**: 마진 최소화(~10%) — 마케팅·광고 비용 절감을 가격으로 환원

## 2. 주요 차별점 / 제품·서비스 구성

### 제품
- 식약처 1일 권장 섭취량을 맞춘 **개인 맞춤 타정 패키지**
- 기능성 인정 성분만 사용, 불필요한 부원료·합성 착향료 최소화
- **1알 약 10,000원** 수준의 개별 단가 (개인화 처방의 가치에 맞춘 프리미엄 포지셔닝)

### 서비스
- **1:1 AI 상담** — 현재 복용 중인 영양제·식단·건강 니즈 분석 → 맞춤 처방 + 과용·중복 경고
- **전용 디스펜서** — 무전원 자판기(뽑기 기계) 형태, 리필 카트리지 교체식
    - 미니멀 디자인: 식탁 위에 놓아도 깔끔
    - 여러 영양제 통을 열 필요 없이 한 번에 섭취 (시간 절약)
    - 개인별 합포장 시스템 대신 디스펜서를 활용해 **개발·공정 난제를 회피**
- **연간 멤버십** (코스트코 모델 벤치마킹) — 가입 시 제품 단가 + 혜택 + 우선 배송

## 3. 시장·경쟁

- **시장**: 건기식 시장 성장 지속, 개인화 수요 증가
- **기존 시장의 문제 (업셀 대화 재료로 활용)**
    1. 광고·마케팅 비용 → 최종 가격 인상
    2. 불필요한 부원료로 제품 부피 증가
    3. 여러 영양제 통을 개별로 여닫는 번거로움 (복용 스트레스 → 섭취 중단 유발)
- **당사 경쟁력**
    - AI 상담으로 **근거 기반 처방** (식약처 고시 인용 + 일일 섭취량 수치 + 과잉 복용 주의)
    - **디스펜서 UX**로 섭취 편의 차별화
    - **멤버십 기반 가격** — 중간 유통·광고비 절감 효과를 소비자에게 환원

## 4. 챗봇·사이트에서의 상담 톤 & 업셀 원칙

챗봇 응답을 설계할 때 항상 사업의 가치 제안과 **정합**하도록 아래 원칙을 따릅니다.

### 4.1 상담 오프닝에 반영할 정보
- 사용자가 **현재 복용 중인 영양제**가 있는지 먼저 묻는 여지 확보 (과용·중복 분석 서비스 암시)
- 우리 서비스의 접근이 "식약처 고시 + 1일 권장 섭취량 최적화"임을 명시

### 4.2 제품 추천 단계에서의 업셀 훅
- 기능성 원료 설명 시 반드시 "식약처 고시 기능·1일 권장 섭취량·과잉 주의" 3종 세트 포함
- 일반 종합비타민 대비 우리 제품은 **부원료 최소화 + 고시 기능 인정 성분만** 사용하는 점을 1줄로 강조 가능

### 4.3 CLOSE/마무리 단계에서의 업셀
- 베이스 제품(데일리 베이스) 제안 후, 아래 2가지 혜택을 같이 안내:
    1. **전용 디스펜서** — 여러 영양제 통을 열 필요 없이 한 번에, 미니멀 디자인
    2. **연간 멤버십** — 코스트코처럼, 가입 시 정기 구독 할인·우선 배송
- "건강을 위해 낭비되는 돈과 시간을 절약한다" 슬로건을 자연스럽게 인용

### 4.4 금지 카피
- 경쟁사 비방
- 질병 치료·예방 효능 단정
- 식약처 고시 외 임의 기능 문구

## 5. 현재 구현 상태 (챗봇 관점)

| 기능 | 구현 여부 | 위치 |
|---|---|---|
| 식약처 고시 기능 인용 + 일일 섭취량 수치 + 과잉 경고 | ✅ | `prompts.py` 공통 규칙 #3 |
| 제품 추천을 표 + 제품별 🛒 구입 버튼으로 제공 | ✅ | `app.py` `_format_products_table` / `_recommendation_actions` |

| 연간 멤버십 가입 플로우 | ❌ 미구현 | 향후: 전용 버튼 + 링크 |
